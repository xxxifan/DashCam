## 产品边界

必须保持：

- 录制必须由用户在前台 UI 明确启动。
- 录制期间必须显示前台服务通知。
- 不实现隐藏、静默、绕过系统限制的摄像头行为。
- 不从完全后台状态偷偷重启摄像头或麦克风。
- 锁屏后继续录制只作为用户前台启动后的系统允许路径。
- 如果系统、用户或 OEM 策略停止服务，App 可以提示用户，但不能绕过平台限制。

当前 v1 不做：

- 手动锁定/保护当前片段。
- 黑屏低亮度备用界面。
- 视频剪辑编辑。
- 投屏 / DLNA。
- TTS、提示音、震动等多通道安全提醒。

## 已确认产品决策

- 包名：`com.xxxifan.dashcam`。
- App 名称：`DashCam`。
- 目标设备基线：Pixel 10a。
- 最低系统：Android 16 / API 36。
- 默认音频：开启。
- 默认分段：2 分钟。
- 默认质量：720p30 标准码率。
- 默认编码器：Auto，由 App 质量策略解析。
- 默认存储策略：使用最大安全可用空间，并保留 10% 存储空间。
- 默认资源压力自动降档：开启。
- 数据持久化：MMKV 保存设置和轻量录制元数据。
- UI 技术：Jetpack Compose + Material 3。
- 录制 API：CameraX，必要时使用 Camera2 interop。
- 播放：Media3 / ExoPlayer。
- 分享：FileProvider。
- 导出：MediaStore 复制到公共媒体目录。

## 架构现状

主要模块：

- `MainActivity.kt`
  - Compose 主界面。
  - 录制首页、视频库、设备页、设置页、播放页。
  - 处理录制启动确认、设置变更、导出、分享、播放。

- `device/remote/DeviceProtocol.kt`（v1.1）
  - 定义 `DeviceProtocolDriver`、`DeviceSession` 和 `NewDeviceSdkProbe`。
  - 预览、远端媒体、下载格式和后处理跟随设备会话，不在 UI 或通用仓库中硬编码。

- `device/remote/DeviceManager.kt`（v1.1）
  - 维护设备驱动注册表、探测、活动会话和设备 UI 状态。
  - 固定通过 Wi-Fi `Network` 探测局域网，设备会话期间绑定进程路由，关闭会话后恢复默认网络。
  - 应用入口、Wi-Fi 网络出现、设备页打开和手动操作均可触发探测。

- `device/remote/Legacy254DeviceDriver.kt`（v1.1）
  - `192.168.1.254` Novatek XML / HTTP / RTSP 协议。
  - 负责预览模式命令、XML 文件列表、设备文件 URL、下载和原始文件保留策略。

- `device/remote/Legacy169DeviceDriver.kt`（v1.1）
  - `192.168.169.1` JSON / HTTP / RTSP TCP 协议。
  - 负责录像/回放模式命令、JSON 文件列表、分页入口、下载和原始文件保留策略。

- `device/remote/DeviceDiagnosticsLogger.kt`（v1.1）
  - 写入 `files/device_logs/device-events-YYYYMMDD.log`。
  - UTF-8 JSON Lines，保留 30 天；设备页显示最近记录并可通过 FileProvider 分享完整文件。

- `device/remote/DeviceScreen.kt`（v1.1）
  - 设备探测结果、新款 SDK 集成状态、设备连接和协议操作 UI。
  - Media3 RTSP 实时预览、远端媒体播放、断点下载进度和设备诊断文件入口。

- `recording/RecordingService.kt`
  - 前台录制服务。
  - CameraX 绑定、开始/停止、分段轮转。
  - 处理 CameraX finalize/error。
  - 执行安全策略、自动降档、异常停止提示。

- `recording/RecordingState.kt`
  - 录制 UI 状态总线。
  - 当前状态只适合单进程轻量使用，未来如需恢复更复杂状态可再拆。

- `storage/LoopStorageManager.kt`
  - 录制目录创建。
  - 空间检查。
  - 循环删除旧片段。

- `storage/RecordingStorageEstimator.kt`
  - 估算录制目录大小、剩余空间、可录时长、下一片段大小。
  - 根据实时录制 stats、历史文件或启发式码率估算。

- `data/RecordingRepository.kt`
  - MMKV 录制元数据。
  - 扫描目录、列表、删除、导出状态更新。

- `data/RecordingSettingsStore.kt`
  - MMKV 设置持久化。

- `data/RecordingAlertStore.kt`
  - 最近一次异常停止原因持久化。
  - 解决服务销毁后警告被普通“录制已停止”覆盖的问题。

- `data/RecordingThumbnailManager.kt`
  - 录制完成后的缩略图生成、缓存和清理。

- `data/RecordingEventLogger.kt`（v1.1 计划）
  - 写入 App 独立录制事件日志文件。
  - 用于长时间录制、循环删除、停止原因和导出复盘。
  - 不依赖 logcat，不要求先做日志页面。

- `camera/CameraCapabilitiesRepository.kt`
  - 查询分辨率、帧率、编码器、HDR、防抖、镜头等能力。

- `camera/PreviewController.kt`
  - 首页预览绑定。

- `safety/RecordingSafetyPolicy.kt`
  - 安全等级、原因、动作和策略接口。
  - 保持和 UI/通知/TTS 等输出通道解耦。

### Phase 8：v1.1 开发候选

状态：进行中；原计划优化和外部记录仪首版支持已落地。

优先级较高：

- [x] 独立录制事件日志文件。
  - 不做日志页面作为前置要求。
  - 日志必须落到 App 私有文件，方便后续复盘和导出。
  - 不能只依赖 logcat。
  - 实际路径：`files/recording_logs/recording-events-YYYYMMDD.log`。
  - 建议格式：JSON Lines，每行一个事件，便于追加、检索、导出和脚本分析。
  - 需要记录：session start/stop、segment start/finalize、CameraX finalize error、循环删除、空间检查、自动降档、安全决策、手动停止、异常停止、导出/分享关键结果。
  - 日志写入必须在 IO 线程，避免阻塞录制主路径。
  - 日志文件需要轮转或保留策略，避免日志无限增长。
  - 后续可以增加“导出诊断日志”入口，但不作为第一步。
- [x] 更明确的停止原因分类：手动停止、App 安全停止、CameraX 错误、系统中断、权限丢失、设备温度、电量。
- [x] 降低录制中 UI 刷新频率。
  - 录制时剩余空间、录制文件大小、设置里的循环录制空间不需要实时刷新。
  - 录制中可暂停设置页存储估算刷新，或降低到较低频率。
  - 首页录制中大小显示用整数粗粒度单位，并节流刷新。
  - 列表顶部录制状态只显示必要状态，避免因字节级变化频繁重组。

## 已知风险

- 部分设备或系统版本可能在锁屏后中断相机，即使前台服务仍在。
- CameraX 能力查询通过不代表所有功能组合都稳定。
- 4K60 高码率会快速消耗空间并触发发热或编码器压力。
- 系统或用户仍可以停止前台服务。
- App 私有外部目录会随 App 卸载删除。
- 导出到 MediaStore 的副本会按设计出现在用户可见媒体库中。
- 两个旧款记录仪协议来自目标 APK 的静态分析，不同固件可能改变 XML/JSON 字段、RTSP 编码或模式切换顺序。
- 旧款记录仪使用局域网明文 HTTP / RTSP，应用为此允许 cleartext 流量；相关能力只应在用户主动连接的记录仪 Wi-Fi 中使用。
- 新款 iCatch 协议依赖未集成的私有 SDK 和 native 库，当前只能记录未集成状态，不能判断真实设备支持性。

## 下一步建议

最近开发重点按顺序推进：

1. 连接 `192.168.1.254` 和 `192.168.169.1` 真机，使用设备诊断文件核对探测、模式切换、RTSP 和文件列表。
2. 根据真实固件样本补充 XML/JSON 兼容解析、RTSP 传输回退和设备专用后处理。
3. 获得合法 iCatch SDK 后实现新款 SDK 探测和设备驱动，不直接依赖旧 APK 私有二进制发布。
4. 继续实现设置组合兼容性保护和视频库大量文件优化。
