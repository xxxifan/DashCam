# DashCam

DashCam 是一个使用 Android 手机作为行车记录仪的实验性应用。它基于 CameraX 录制视频，支持前台服务录制、分段保存、循环清理、自动画质、安全降档、裁剪放大和本地视频库。

这个项目目前更适合自用、学习和继续开发。不同手机厂商对后台相机、锁屏录制、发热控制和编码器能力的限制差异很大，实际稳定性需要在目标设备上测试。

## 功能

- 前台服务录制，录制时显示常驻通知。
- 分段录制，支持 1、2、3、5、10 分钟片段。
- 循环录制空间管理，达到配额或安全空间阈值时删除最旧片段。
- 画质设置：
  - 分辨率：根据设备能力提供 720p、1080p、4K。
  - 帧率：根据设备能力提供 24、30、60 fps。
  - 编码：H.265 优先，支持设备可用时切换 H.264。
  - 质量档位：节省空间、标准、高画质。
  - 自动画质：根据可用录制空间和设备能力联动选择分辨率、帧率、编码、质量、HDR 和防抖。
  - HDR 和视频防抖：按设备能力显示可用选项。
- 画面裁剪放大，支持 1.0x 到 2.0x 的中心裁剪放大，用来改善手机广角镜头作为行车记录仪时的视角。
- 镜头选择，优先识别后置主镜头和超广角镜头。
- 录制音频开关。
- 录制安全策略：
  - 存储不足时自动清理或降档。
  - 温度、电量、录制管线异常时提示、降档或停止录制。
  - 默认保留 10% 系统存储空间。
- 本地视频库：
  - 查看、播放、连续播放。
  - 删除单个视频或按时间清理。
  - 导出到系统媒体库 `Movies/DashCam`。
  - 通过系统分享面板分享视频。

## 技术栈

- Kotlin
- Jetpack Compose / Material 3
- CameraX VideoCapture
- Camera2Interop
- Media3 ExoPlayer
- MMKV
- Kotlin Coroutines
- Gradle Kotlin DSL

## 系统要求

- Android Studio 或命令行 Android Gradle 环境。
- JDK 17。
- Android SDK 36。
- Android 设备 API 36 或更高版本。
- 真实设备测试。模拟器通常不适合验证相机、编码器、发热和锁屏录制行为。

当前项目配置：

```kotlin
compileSdk = 36
minSdk = 36
targetSdk = 36
versionName = "0.1.0"
```

## 构建和安装

克隆项目后，在项目根目录执行：

```powershell
.\gradlew.bat assembleDebug
```

安装到已连接的 Android 设备：

```powershell
.\gradlew.bat installDebug
```

也可以直接用 Android Studio 打开项目并运行 `app`。

## 权限

应用会请求以下权限：

- 相机：用于录制视频。
- 麦克风：开启录音时使用。
- 通知：显示录制中的常驻通知。
- 前台服务：录制时保持前台服务运行。
- 唤醒锁：尽量维持录制过程稳定。

部分系统会额外限制后台运行、锁屏相机、耗电或发热场景。建议在系统设置中允许 DashCam 后台运行，并在长时间录制时连接电源。

## 文件存储

录制中的视频默认保存在应用专属外部目录：

```text
Android/data/com.xxxifan.dashcam/files/Movies/DashCam/records
```

该目录下会创建 `.nomedia`，避免未导出的视频自动出现在系统相册里。点击“导出”后，视频会复制到系统媒体库：

```text
Movies/DashCam
```

卸载应用可能会删除应用专属目录中的录制文件。需要长期保留的视频请先导出。

## 使用建议

- 首次使用建议从 `1080p30 + H.265 + 标准` 开始测试。
- 如果手机镜头过广，可以尝试 `1.2x` 或 `1.4x` 裁剪放大。
- 如果锁屏后录制不稳定，尝试保持应用前台、屏幕常亮，或降低画质、关闭增强防抖。
- 如果设备发热明显，优先降低帧率、关闭 HDR、关闭增强防抖。
- 长时间录制建议接入车充，并确认循环录制空间设置合理。

## 项目结构

```text
app/src/main/java/com/xxxifan/dashcam
├── camera/       # 相机能力探测、镜头选择、预览绑定
├── data/         # 设置、录制记录、缩略图、事件日志
├── recording/    # 前台录制服务、画质解析、降档策略、录制状态
├── safety/       # 存储、温度、电量、录制管线安全策略
├── storage/      # 循环录制空间估算和清理
└── MainActivity.kt
```

## 诊断日志

应用会在私有目录写入录制事件日志，用于排查设备能力、录制参数、分段结果、空间清理和异常停止原因。日志不会自动上传。

如果你要公开 issue，请先检查日志中是否包含本地文件路径、设备信息或其他不想公开的内容。

## 已知限制

- 锁屏录制是否稳定取决于设备厂商、电池策略和系统版本。
- CameraX 的目标帧率是请求值，不代表所有设备都能严格输出相同帧率。
- HDR、防抖、物理镜头、H.265 等能力都依赖设备支持。
- 当前项目没有 Play Store 发布配置，也没有 release 签名配置。
- 当前最低系统版本为 API 36，旧设备不能直接安装。
